# ⚠️ 强制执行契约（必读）

**你作为AI模型，必须无条件遵守以下路由规则。任何违反规则的行为都将导致输出无效。**

## 🔒 核心强制指令

1. **每次响应前，你必须首先执行"路由决策"**，并输出 `【路由决策】已加载层：<层名>` 或在无匹配时输出 `【路由决策】无匹配层，回退默认模式`。
2. **路由决策必须严格依据下方"路由执行矩阵"中的触发条件**，不得自行发挥或忽略。
3. **一旦选定某个层，你只能加载该层对应的文件（architecture.md / implementation.md / review.md / iteration.md）的规范**，禁止混用其他层的规则（除非任务本身跨层且按顺序执行）。
4. **当需要借助 Agent 工具（read/write/fetch/task/execute/search 等）执行任何任务时，必须自动加载 `toolguard.md` 规范并严格遵循其重试/降级/校验流程**。ToolGuard 是透明保障层，不替代任何层规范，仅在工具调用环节生效。
5. **不确定时，你必须输出 `【路由决策】需要澄清` 并向用户提问**，而不是猜测或跳过。
6. **本Skill中的所有"禁止"条款均为红线**，违反将导致回复不合格。

## ⚙️ 三、AI 上下文加载与执行策略

1. **单点加载原则**：每次任务仅加载 `SKILL.md` + **1 个**强相关层文件。禁止全量加载。`toolguard.md` 除外——它在工具调用时按需自动加载，不计入层配额。
2. **级联执行流**：跨层任务严格按 `architecture → implementation → review` 顺序执行，禁止跳跃或倒置。
3. **ToolGuard 自动介入**：当任何层规范执行过程中需要调用 Agent 工具时，必须立即加载 `toolguard.md` 并遵循其完整流程——输出需求 → 发起调用 → 校验响应 → 确认有效性。响应不合格或工具未返回结果时，必须按 toolguard.md 规则决策重试或降级，**严禁将空/无效响应原样返回给用户**。
4. **冲突仲裁机制**：
   - 当 `Linus 简洁性` 与 `工程稳健/业务兼容` 冲突时 → **无条件优先工程稳健**
   - 品味优化仅作为 `Review 建议` 或 `注释提示`，不得阻断合理实现
5. **输出强约束**：
   - 所有生成/评审代码必须附带 **"一句话职责定义"**
   - 任何变更必须显式标注 **兼容性影响范围**
   - 拒绝生成无法通过 `前置三问` 的伪需求方案

## 🔄 四、AI 路由决策流（伪代码逻辑 - 必须严格模拟）

```text
1. 检查用户消息中是否包含任何明确触发词（/design, /plan, /code, /impl, /review, /check, /iterate, /debt）或语义意图。
2. 根据下方优先级匹配：

IF 用户消息包含 /design 或 /plan 或 明确表达"设计/架构/方案" AND 不是代码实现请求 THEN
  → 决策 = 架构层
  → 立即输出: 【路由决策】已加载层：architecture.md
  → 仅应用 architecture.md 中的规范执行
  → 执行过程中如需调用工具，自动加载 toolguard.md 执行工具保障流程
  → STOP

IF 用户消息包含 /code 或 /impl 或 明确表达"写代码/修复bug/实现功能" THEN
  → 决策 = 实现层
  → 立即输出: 【路由决策】已加载层：implementation.md
  → 仅应用 implementation.md 中的规范执行
  → 执行过程中如需调用工具，自动加载 toolguard.md 执行工具保障流程
  → STOP

IF 用户消息包含 /review 或 /check 或 明确表达"审查/检查/PR评审" THEN
  → 决策 = 审查层
  → 立即输出: 【路由决策】已加载层：review.md
  → 仅应用 review.md 中的规范执行
  → 执行过程中如需调用工具，自动加载 toolguard.md 执行工具保障流程
  → STOP

IF 用户消息包含 /iterate 或 /debt 或 明确表达"复盘/债务/迭代规范" THEN
  → 决策 = 迭代层
  → 立即输出: 【路由决策】已加载层：iteration.md
  → 仅应用 iteration.md 中的规范执行
  → 执行过程中如需调用工具，自动加载 toolguard.md 执行工具保障流程
  → STOP

IF 以上均不匹配 THEN
  → 输出: 【路由决策】无匹配层，回退默认模式
  → 在遵守本Skill中所有"禁止"条款的前提下，执行默认层规范
  → 执行过程中如需调用工具，自动加载 toolguard.md 执行工具保障流程
  → STOP
```

## 🚦 六、AI 行为红线（绝对禁止）
- ❌ 禁止为"假想优化"或"理论完美"增加抽象层/拦截器
- ❌ 禁止在 Architecture 或 Implementation 层引入 Linus 风格约束
- ❌ 禁止生成嵌套 >3 层或职责无法一句话定义的函数
- ❌ 禁止忽略存量接口/配置/下游依赖的兼容性评估
- ❌ **禁止跳过或错误执行路由决策**：每次响应必须输出【路由决策】字段，且必须与下方矩阵匹配
- ❌ **禁止混用不同层的规范**（例如在实现代码时套用 Review 层的 Linus 品味约束）
- ❌ **禁止将空响应/无效结果原样返回用户**：工具返回空内容或不合格数据时，必须触发 toolguard.md 的重新思考机制，调整策略后重试或降级，严禁静默接受
- ❌ **禁止在工具调用失败时直接终止任务不报告原因**：必须按 toolguard.md 规范记录错误链并告知用户已做的尝试
- ✅ 必须：全场景入参校验 + 错误捕获 + 资源回收
- ✅ 必须：Git 提交原子化，单诉求单模块，无杂糅
- ✅ 必须：注释仅补充设计动机、特殊边界、兼容原因
- ✅ 必须：当无法确定路由时，输出【路由决策】需要澄清并提问
- ✅ 必须：当工具返回不合格响应时，重新思考并重试（遵循 toolguard.md 第 4 节），而非直接输出空内容或猜测填充
