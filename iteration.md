# 🔄 迭代层规范
> 定位：能力固化引擎｜周期复盘｜规范自演进｜工具链闭环

## 1. 时间维度迭代机制
### 📅 当日迭代（5-10 分钟）
- 复盘当日编写最糟糕的一段逻辑
- 对照主体规范 + Linus Review 约束做精简重构
- 记录一条当日代码风格问题至 `personal_debt.md`

### 📆 本周迭代（30 分钟）
- 批量复盘 GitHub 仓库近期全部提交
- 统计高频重复问题，形成个人避坑清单 `mogumc-antipatterns.md`
- 锁定单周单一优化目标（嵌套/分支/函数拆分任选其一，轮换执行）

### 📆 本月迭代（1-2 小时）
- 选取历史老旧模块（修改频次高或含多处 Review 警告）做轻量化重构
- 统一零散编码习惯，收敛为固定范式
- 更新本地 Review 检查清单，输出重构前后对比与收益量化

## 2. 工具层强制约束
- **格式化工具**：统一代码样式（Go: `gofmt`/`goimports`, Frontend: `prettier`）
- **Lint 拦截**：静态分析拦截低级风格与语法问题，阻断提交
- **Commit Gate**：提交前自行快速 Review 自检（运行自定义 `make review-check`）
- **自动化门禁**：无 🟡/🔴 未处理警告 + Changelog 变更声明 → 才允许 Push

## 3. 规范演进与能力固化
- **个人风格数据集**：收集高频重复模式、错误包装风格、参数校验写法 → 沉淀为 `mogumc-style.md` 团队 Onboarding 文档
- **规范触发更新机制**：同类问题在代码库中出现 ≥ 3 次 → 强制触发规范版本迭代（例：嵌套超 2 次 → 新增「必须使用策略模式」条款）
- **反馈闭环**：来源 = [Linus 评审警告, 线上问题复盘, 协作反馈] → 动作 = 季度回顾 → 输出 = `Skill Architecture v1.x` 更新
- **AI 执行适配**：所有层规则已结构化，可直接注入 LLM System Prompt 或 IDE Agent，实现上下文感知的自动审查与重构建议
